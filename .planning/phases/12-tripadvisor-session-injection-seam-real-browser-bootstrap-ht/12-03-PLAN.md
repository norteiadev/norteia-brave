<objective>
Refactor TripAdvisorClient for the session-injection model: introduce SessionMissingError,
make _get_session() Redis-only (no auto-bootstrap), fix the persisted-query payload to
the correct batch-array + extensions.preRegisteredQueryId shape, and remove the Playwright
_bootstrap_session path plus the scraper optional dep group.

Purpose: The Phase 11 client has the wrong GraphQL payload format ({"query": queryId}
instead of {"extensions": {"preRegisteredQueryId": "..."}}). This is the core fix that
makes real TripAdvisor fetches work. Without this, every POST to graphql/ids returns
an error even with valid DataDome cookies.

Output: client.py without _bootstrap_session or ThreadPoolExecutor; correct payload shape;
SessionMissingError raised on Redis miss; pyproject.toml without the scraper dep group;
updated test_client.py with new offline test coverage and real_browser marker class removed.
</objective>

<!-- Key payload shape confirmed by spike 2026-06-24 -->

Real TripAdvisor persisted-query POST body (what the server actually expects):
```json
[
  {
    "variables": {"locationId": 303513, "offset": 0, "limit": 20},
    "extensions": {"preRegisteredQueryId": "a1b2c3d4e5f6a7b8"}
  }
]
```

Phase 11 client sends (WRONG — must be replaced):
```json
[
  {
    "query": "a1b2c3d4e5f6a7b8",
    "variables": {"locationId": 303513, "offset": 0, "limit": 20}
  }
]
```

Session dict shape written by the injection endpoint (plan 12-02) and read by _get_session:
```json
{
  "cookies": {"datadome": "abc", "TASession": "xyz"},
  "query_ids": {"destinations": "a1b2c3d4e5f6a7b8", "attractions": "b2c3d4e5f6a7b8c9"},
  "user_agent": "Mozilla/5.0 ...",
  "acquired_at": "2026-06-24T12:00:00Z"
}
```

Note: phase 11 stored cookies as a list [{name, value, domain}] — the injection endpoint
(plan 12-02) stores them as a flat dict. The refactored _get_session must handle BOTH shapes
for backwards compatibility: if cookies is a list, convert to {c["name"]: c["value"]} dict.

| fetch_* → TripAdvisor graphql/ids | Production httpx POST; cookies passed from Redis-stored session |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-12-03-02 | Denial of Service | SessionMissingError propagation — uncaught in caller raises unhandled exception | mitigate | Plan 12-04 adds catch in sweep_tripadvisor; the endpoint (plan 12-02) validates the session before workers can read it; SessionMissingError is a clean, catchable error type |
| T-12-03-03 | Information Disclosure | user_agent header added to httpx requests | accept | user_agent comes from the operator-injected session body; it is the real browser UA used by the human operator; mirroring it is intended and not a disclosure issue |
