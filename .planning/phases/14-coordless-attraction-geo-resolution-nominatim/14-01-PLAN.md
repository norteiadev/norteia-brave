<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| collector → nominatim.openstreetmap.org | Outbound GET to a public, keyless third-party endpoint; response is untrusted JSON |
| geocode result → Redis cache | Attacker-controlled locationId as cache key; malformed result shape injected via cache |
| Redis cache → atrativos._ingest_one | Cached geo dict consumed as trusted in Wave 2; shape must be validated on read |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Information Disclosure (LGPD) | nominatim.py geocode result dict + Redis cache | mitigate | Result dict limited to exactly 4 keys (lat, lon, osm_id, municipio_name); structlog never receives name or raw address object; setex stores only the 4-key dict — address.street/postcode/display_name never persisted (locked CONTEXT decision #8) |
| T-14-02 | Tampering | nominatim.py JSON response parsing | mitigate | Defensive .get() chains for all address sub-keys; float() conversion for lat/lon raises on non-numeric; empty data list → None + cache sentinel (never raises unguarded KeyError) |
| T-14-03 | Denial of Service (self-inflicted) | nominatim.py rate limit + cache | mitigate | min_request_interval=1.1 s enforced via asyncio.sleep per Nominatim policy (≤1 req/s); cache by locationId with 30d TTL + negative sentinel prevents repeated same-query ToS violation; identifiable User-Agent required by policy |
| T-14-04 | Tampering (cache poisoning) | Redis key brave:geo:{location_id} | accept | locationId comes from TripAdvisor AttractionsFusion response (already behind session auth); format is a TA integer string; no cross-user cache sharing in this operator-gated sweep; risk is low — cache is safe to flush |
| T-14-05 | Spoofing (SSRF) | nominatim.py _NOMINATIM_SEARCH_URL | mitigate | Base URL pinned as a module constant "https://nominatim.openstreetmap.org/search" overridable only via BRAVE_NOMINATIM_BASE_URL config (not from user input); httpx enforces HTTPS; no URL construction from untrusted data |
| T-14-SC | Tampering | npm/pip/cargo installs | accept | Phase 14 installs no new packages (Package Legitimacy Audit: not applicable per 14-RESEARCH.md); all dependencies pre-vetted at project setup |
</threat_model>
