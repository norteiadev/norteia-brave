## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DesmembramentoAgent → LLMClientProtocol | LLM-generated content enters Nascente; 2nd-layer validator is the gate |
| Mtur seed content → LLM prompt | Município names from seed CSV are embedded in the DeepSeek prompt |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08-01 | Tampering | Prompt injection via Mtur município names | mitigate | Mtur CSV is a trusted bundled dataset (public Mtur data). Names go into a structured prompt with explicit instruction set. The 2nd-layer DesmembramentoResult Pydantic validator rejects output that doesn't match the schema. Quarantine handles instructor retry exhaustion — no unvalidated data reaches Nascente. |
| T-02-08-02 | Information Disclosure | DeepSeek sees município names | accept | All data sent to DeepSeek via OpenRouter is public geographic data. provider.data_collection="deny" is enforced in the real LLMClient implementation (D-10 of Phase 1 LLM config). The fake client in Phase 2 tests never sends data externally. |
| T-02-08-03 | Elevation of Privilege | origem=40 firewall bypass | mitigate | D-06 firewall is a pure scoring consequence (max score 67 without validacao_humana). No code branch possible. Verified by test_producer_score_boundaries (Plan 05). |
| T-02-08-04 | Denial of Service | Unbounded LLM fan-out (all Oferta Principal for a large UF) | mitigate | Fan-out is bounded by the number of Oferta Principal municipalities per UF (hundreds, not thousands). The cost guard in LLMConfig.usd_daily_budget enforces a spending ceiling. The real LLMClient (Phase 3) checks the guard before dispatch. |
| T-02-08-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages. slopcheck not required. |
</threat_model>
