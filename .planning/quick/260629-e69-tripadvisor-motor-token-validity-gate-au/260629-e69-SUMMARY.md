# Phase quick-260629-e69 Plan 01: TA Motor Token Validity Gate Summary

## One-liner

Replaced 260628-m1n auto-resume machinery with R1 engine-off-on-expiry + R2 session-validity pre-flight gate — no auto-restart ever fires; operator must inject a fresh session before the motor can start.

## What Was Built

**Part A — Remove auto-resume machinery (260628-m1n):**
- Deleted `brave/lanes/tripadvisor/resume.py` (`maybe_resume_bulk_sweep`)
- Deleted 3 test files for the removed machinery
- `sweep_progress.py`: removed `RESUMING` constant, `_RESUME_CLAIM_KEY`, `_F_DEPTH/_F_GEO_ID/_F_TARGET_MAX_PAGES`, `is_paused_needs_bootstrap()`, `claim_resume()`, `get_resume_params()`; reverted `start()` to simple 2-arg signature
- `tripadvisor_session.py`: removed `maybe_resume_bulk_sweep` import and auto-resume block; removed `"resuming"` from `TASweepProgressResponse.state` Literal
- `pipeline.py`: removed `ta_resume_watch` task; reverted `sweep_progress.start()` extra kwargs
- `beat_schedule.py`: removed `ta-resume-watch` beat entry

**Part B — R1: session expiry turns engine OFF:**
- `pipeline.py` except block: after `stop_needs_bootstrap(rc)`, calls `collection_engine.set_enabled(rc, False)` + `collection_engine.mark_idle(rc)`
- Per-UF path (rc=None) uses `redis.from_url(BRAVE_DB_REDIS_URL)` for R1 (patched to fakeredis in tests)

**Part C — R2: engine start requires live TA session:**
- `engine.py` `engine_start()`: gate before `start_run` for `source='tripadvisor'`; `redis.ttl(BRAVE_TA_SESSION_KEY)` — if `ttl != -1 and ttl <= 0` → 409 with PT-BR detail
- TTL -1 (no expiry, operator set manually via redis-cli) accepted as infinite session

**Part D — Dashboard PainelTopbar gate:**
- `taBlocked = source === "tripadvisor" && (!sessionStatus?.present || expires_in <= 0)`
- `onToggleMotor`: `toast.error(...)` + `return` when `taBlocked` (depth menu never opens)
- `explainError`: 409 now returns `err.message` (backend detail) not hardcoded string
- Auto-off `useEffect`: `toast.warning(...)` when `enabled` transitions `true → false` mid-run
