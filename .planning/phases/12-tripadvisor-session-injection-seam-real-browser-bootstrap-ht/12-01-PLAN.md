  <name>Task 2: scripts/ta_bootstrap entry point + parser</name>
  <files>scripts/ta_bootstrap, scripts/ta_bootstrap.py</files>
  <action>
    Create scripts/ta_bootstrap as a thin executable shell wrapper (chmod +x):

      #!/usr/bin/env bash
      # TripAdvisor session-injection bootstrap helper (TA-09)
      # Usage: ./scripts/ta_bootstrap [--curl FILE] [--endpoint URL] [--bearer TOKEN]
      exec python "$(dirname "$0")/ta_bootstrap.py" "$@"

    Create scripts/ta_bootstrap.py with these capabilities:

    CLI arguments (argparse):
      --curl FILE     Path to a file containing a pasted cURL string. If omitted,
                      the script reads from stdin (operator pastes the cURL interactively).
      --endpoint URL  Base URL of the FastAPI service (default: http://localhost:8000).
      --bearer TOKEN  Bearer token for auth (env: BRAVE_DASHBOARD_BEARER_TOKEN if omitted).

    Parsing logic (function parse_curl(curl_str: str) -> dict):
      Given a "curl 'URL' -H '...' --data-raw '...'" string from DevTools Copy-as-cURL:
      1. Extract cookies: find the -b / --cookie or -H 'Cookie: ...' section; split into
         {name: value} dict. The datadome cookie and TA session cookies (__utma, TASession, etc.)
         are all in the Cookie header.
      2. Extract user_agent: find -H 'User-Agent: ...' or -A '...' in the curl string.
      3. Extract query_ids: parse the --data-raw JSON (which is a BATCH ARRAY — list of dicts);
         for each item in the array, read item["extensions"]["preRegisteredQueryId"] if present.
         Collect unique queryIds into {"destinations": <first_found>, "attractions": <second_found>}
         or use heuristic: if variables contain "ATTRACTION" (case-insensitive) → attractions key,
         else → destinations key. If the cURL only contains one request, store it under both keys
         as a safe fallback (operator can re-capture a second query for the other entity type).
      4. acquired_at: datetime.utcnow().isoformat() + "Z".
      5. Return a dict matching the POST /api/v1/tripadvisor/session Pydantic body:
         {
           "cookies": {name: value, ...},   # flat dict
           "query_ids": {"destinations": ..., "attractions": ...},
           "user_agent": "...",
           "acquired_at": "...",
         }

    Injection logic (function inject_session(payload: dict, endpoint: str, bearer: str) -> None):
      POST {endpoint}/api/v1/tripadvisor/session with Authorization: Bearer {bearer} header
      and Content-Type: application/json. Use only stdlib (urllib.request, json) — no requests lib.
      On 200/201/202: print "Session injected — canary result: {response_body.get('canary')}".
      On 422: print "Validation error — check cookies/query_ids" + response body.
      On 4xx/5xx: raise SystemExit with the status + body.

    main() function:
      1. Parse args.
      2. Read curl string from --curl file or stdin.
      3. Call parse_curl → payload dict.
      4. Print "Parsed: {cookie_count} cookies, query_ids={...}".
      5. bearer = args.bearer or os.environ.get("BRAVE_DASHBOARD_BEARER_TOKEN", "")
      6. inject_session(payload, args.endpoint, bearer).

    The script must have no third-party imports (only stdlib: argparse, json, re, os, datetime,
    urllib.request). This keeps it runnable before the venv is activated.

    The cookies field sent to the API is a flat dict (not a list of {name, value, domain} dicts).
    The endpoint accepts both shapes — but the script sends the flat dict form for simplicity.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
