# Phase 17.1 Plan 07: Origem Modal + Motor Depth + Shell Integration Summary

The integration plan that finishes the Painel Brave shell: the **Origem** source-pick modal with TripAdvisor cURL (re)inject (distinct 422/503 handling + real TTL), the **depth-required Motor** toggle, the real-`expires_in` TA pill, and the page **view-switcher** that makes all six views reachable in one shell — legacy dark routes untouched.

## What Was Built

**Task 1 — PainelOrigem.tsx modal + test (commit `710d39d`)**
- `dashboard/components/painel/PainelOrigem.tsx` (`"use client"`): radio rows for `mtur` / `tripadvisor` / `google_places` (design copy + descriptions). Selecting TripAdvisor reveals a cURL textarea; an in-modal `parseTACurl()` extracts the cookie jar (`-H 'cookie: …'` / `-b`), `user-agent`, and `preRegisteredQueryId` ids into the strict `SessionInjectBody` shape, stamps `acquired_at` at submit, and posts through `injectTASession` (useMutation). `ApiError.status === 422` ("sessão inválida — cole um cURL atual") is surfaced distinctly from `503` ("verificação indisponível — tente novamente") as both a sonner toast AND an inline `origem-error-422` / `origem-error-503` state. A live TTL badge reads `fetchTASessionStatus().expires_in` (amber ≤5 min). Pure `--painel-*` token styling.
- `__tests__/PainelOrigem.test.tsx`: 6 offline Vitest — source radios render; closed renders nothing; textarea revealed only after TripAdvisor; a cURL paste submits the parsed `{cookies, query_ids, user_agent, acquired_at}` body to `injectTASession`; a 422 handler surfaces the invalid-session state; a 503 handler surfaces the infra state (distinct, mutually exclusive).

**Task 2 — PainelTopbar.tsx (commit `bc86727`)**
- Made the "Origem {source}" button clickable → opens `PainelOrigem` (preselecting the live engine source). Changed the motor toggle so START opens a depth menu (`DEPTH_LABELS`, 3 options) and `onPickDepth` calls `startEngine({ depth })` — a depthless start is impossible (mirrors the backend 422 twin); STOP still calls `stopEngine()`. Drove the TA pill label/color off the real `expires_in` (`sessionWarning` ≤5 min → amber + "Expira em m:ss"), plus a one-shot expiry toast guarded by a ref so the 10s poll can't spam it.
- `__tests__/PainelTopbar.test.tsx` updated for the new flow: idle toggle opens the depth menu (no start without a depth); picking `nascente_rio` fires POST `/start` with `{ depth: "nascente_rio" }`; opening the menu without picking never starts; the pill warns from `expires_in: 120`. 10 green.

**Task 3 — Two-group nav + page view-switcher (commit `05a2dbc`)**
- `dashboard/app/painel/page.tsx`: the view-switcher (now a `PainelBody` switch) renders `PainelDuplicados` and `PainelVarreduras` for their nav keys alongside the existing Painel/Mapeamento/Conversas/Custo — the "Em breve" placeholder is dropped (all six views real). The record-edit Drawer stays reachable from a board card inside `PainelView`.
- `dashboard/components/painel/PainelSidebar.tsx`: added a `GROUP_TESTID` map (pins the two literal group labels Processamento/Operação) and stable per-group + per-item `data-testid`s. The two-group nav itself was already centralized in `nav.ts` since slice 1.

## Deviations from Plan

**1. [Rule 1 — Test alignment] Updated PainelTopbar.test.tsx for the new motor-start flow**
- **Found during:** Task 2.
- **Issue:** The plan's `files_modified` listed only `PainelTopbar.tsx`, but Task 2's required behavior change (START now collects a depth instead of `window.confirm`) invalidated the two existing `window.confirm`-based start tests, which would have failed the suite.
- **Fix:** Rewrote those two tests to the depth-menu flow (open menu → pick depth → assert `{depth}` body; open menu without picking → no start) and added a pill-warn-from-`expires_in` test. This is a test-only change matching the intended new behavior, not a scope expansion.
- **Files modified:** `dashboard/components/painel/__tests__/PainelTopbar.test.tsx`.
- **Commit:** `bc86727`.

Both mitigations in the plan's threat register (T-17.1-07-01/02/03) are honored: distinct non-silent 422 surfacing, no depthless start. No threat flags.
