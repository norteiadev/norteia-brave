<objective>
Implement TripAdvisor session auto-refresh so the operator pastes a cURL only once.

Two building blocks:
  A. Cookie write-back — a single `persist_rotated_cookies()` helper (brave/lanes/tripadvisor/session.py)
     that merges response Set-Cookie back into brave:ta:session and slides the TTL.
     Wired to all 3 fetch transports in client.py. Best-effort / non-fatal.
  B. Keep-alive beat — a new `brave.ta_keepalive` Celery task that fires every ~10 min
     (when session TTL>0 and run_real_externals=True), issues one light HTML GET via
     fetch_attractions_paginated(max_pages=1) to re-mint datadome, and delegates
     write-back to the same shared helper.

Purpose: Prevent session staleness between sweeps by maintaining a sliding TTL.
Output: session.py (new), client.py (modified), settings.py (1 field), pipeline.py
        (new task), beat_schedule.py (beat entry), 2 new offline test files.
</objective>

    Logic:
    1. If response_cookies is empty → return immediately (no Redis write)
    2. raw = redis.get(BRAVE_TA_SESSION_KEY); if None → return (session gone)

    6. If "TASID" in response_cookies → session["session_id"] = response_cookies["TASID"]
    7. redis.setex(BRAVE_TA_SESSION_KEY, ta_config.session_ttl, json.dumps(session))
    8. logger.debug("ta_session_writeback", rotated_cookie_count=len(response_cookies))
       — NEVER log cookie names or values (T-12-02-01)

    Wrap steps 2–8 in try/except Exception as exc:
    - logger.warning("ta_session_writeback_error", error_type=type(exc).__name__)
    - DO NOT reraise — best-effort contract

    @shared_task(
        bind=False,
        max_retries=0,
        name="brave.ta_keepalive",
        ignore_result=True,
    )
    def ta_keepalive() -> None:
        """Keep-alive beat: refresh DataDome cookies when session is live (260629-p2v).

        Fires on a periodic interval (BRAVE_TA_KEEPALIVE_INTERVAL_SECONDS, default 600s).
        Issues ONE light HTML GET via fetch_attractions_paginated(max_pages=1) to re-mint
        datadome + __vt. Cookie write-back is handled inside fetch_attractions_paginated
        (session.persist_rotated_cookies), sliding the session TTL automatically.

        Skips silently when:
          - run_real_externals is False (offline / CI)
          - No session in Redis (brave:ta:session TTL <= 0)

        On 403/SessionExpiredError/SessionMissingError:
          Same fallback as sweep_tripadvisor: sets needs_bootstrap + engine OFF.
          Does NOT crash the beat (exception is caught and logged).

        On any other exception: logs error_type at WARNING and returns. Never raises.

        T-12-02-01: Never logs cookie values (only error_type and ttl_before).
        T-12-04-01: str(exc) is never logged (may contain cookie fragments).
        """
        app_config = AppConfig()
        if not app_config.run_real_externals:
            logger.debug("ta_keepalive_skipped_offline")
            return

        import redis as _redis_lib  # noqa: PLC0415

        _redis_url = os.environ.get("BRAVE_DB_REDIS_URL", "redis://localhost:6379/0")
        rc = _redis_lib.from_url(_redis_url)

        from brave.lanes.tripadvisor.client import BRAVE_TA_SESSION_KEY  # noqa: PLC0415

        ttl = rc.ttl(BRAVE_TA_SESSION_KEY)
        if ttl <= 0:
            logger.debug("ta_keepalive_skipped_no_session")
            return

        from brave.config.settings import TripAdvisorConfig  # noqa: PLC0415
        from brave.lanes.tripadvisor.client import (  # noqa: PLC0415
            SessionExpiredError,
            SessionMissingError,
            TripAdvisorClient,
        )
        from brave.core import engine as collection_engine  # noqa: PLC0415

        ta_config = TripAdvisorConfig()
        ta_client = TripAdvisorClient(config=ta_config, redis=rc)

        try:
            import asyncio as _asyncio  # noqa: PLC0415

            async def _ping() -> None:
                # ONE HTML GET (all-Brazil geoId 294280, page 1) to re-mint datadome.
                # geo_id=294280 is the all-Brazil national listing (same as bulk sweep).
                # fetch_attractions_paginated calls persist_rotated_cookies internally.
                async for _offset, _cards in ta_client.fetch_attractions_paginated(
                    geo_id=294280, start_page=1, max_pages=1
                ):
                    pass  # write-back + TTL slide happened inside; result not needed

            _asyncio.run(_ping())
            logger.info("ta_keepalive_ok", ttl_before=ttl)

        except (SessionExpiredError, SessionMissingError) as exc:
            # Same fallback as sweep_tripadvisor: needs_bootstrap + engine OFF.
            _mark_needs_bootstrap()
            collection_engine.set_enabled(rc, False)
            collection_engine.mark_idle(rc)
            logger.warning(
                "ta_keepalive_session_expired",
                error_type=type(exc).__name__,
                # T-12-04-01: never log str(exc) — may contain cookie fragments
            )

        except Exception as exc:  # noqa: BLE001
            # Unknown error (DNS, proxy, asyncio loop conflict) — log and return.
            # The beat scheduler MUST NOT crash; the next interval fires normally.
            logger.warning(
                "ta_keepalive_error",
                error_type=type(exc).__name__,
            )

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TA response → Redis session | Response Set-Cookie headers from tripadvisor.com are trusted (same-origin cookies); merged into stored session. Attacker cannot inject arbitrary cookies via this path without controlling the TA response. |
| Beat worker → httpx | Keep-alive beat issues outbound GET to tripadvisor.com; response cookies are the only external input merged into session state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-p2v-01 | Information Disclosure | session.py:persist_rotated_cookies log calls | mitigate | Log only rotated_cookie_count (int) and error_type (class name). Never log cookie names or values. Matches T-12-02-01 discipline already enforced in tripadvisor_session.py. |
| T-p2v-02 | Information Disclosure | pipeline.py:ta_keepalive except handlers | mitigate | Log only error_type=type(exc).__name__. Never log str(exc) — same as T-12-04-01 in sweep_tripadvisor. |
