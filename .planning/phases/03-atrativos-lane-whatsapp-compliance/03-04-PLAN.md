## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LangGraph node → WhatsAppClientProtocol | Always via _compliant_send (never direct send_template); compliance gate is the single enforcement point |
| outreach_task → AsyncPostgresSaver | DSN prefix stripped to postgresql:// before passing to langgraph checkpointer; DB credentials in BRAVE_DB_URL env var only |
| inbound message text → DeepSeek extraction | Only the message body (not phone number or PII) is sent to LLM; data_collection=deny per LLMConfig (Phase 1) |
| recv_reply node → opt-out detection | Exact keyword match (no LLM, no eval); keyword logged to consent_log; phone number never forwarded |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-03-04-02 | Tampering | LangGraph thread_id collision (wrong conversation resumed) | mitigate | thread_id = f"atrativo:{rio_id}" (UUID immutable); resume_conversation_task verifies sub_state == "whatsapp_in_progress" before proceeding |
