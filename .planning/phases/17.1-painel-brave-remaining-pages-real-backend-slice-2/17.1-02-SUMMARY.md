# Phase 17.1 Plan 02: Varreduras runs-history backend Summary

Durable `runs_history` envelope written at engine-start and finalized when the sweep idles, plus a `GET /api/v1/runs` list (on-read synced/failed/total) and a `PATCH /runs/{id}/reprocess` scope re-dispatch — closing the gap where engine runs lived only in Redis with no DB audit trail.

## What Was Built

**Task 1 — model + migration (commit 21442b3)**
- `RunHistory(Base)` in `brave/core/models.py` (`runs_history`): id, started_at (indexed, server_default now()), ended_at, ufs (JSON), source/depth/lane, ufs_total/ufs_dispatched, nullable total/synced/failed, status. Mirrors `LLMGeneration` style exactly.
- Alembic `0007_add_runs_history.py` (down_revision `0006`): `create_table` + plain non-CONCURRENTLY `ix_runs_history_started_at` (inside the Alembic txn, mirroring 0006).
- `tests/integration/test_migration_0007.py` — offline revision-metadata test (revision `0007`, down_revision `0006`) + skip-safe `@pytest.mark.skipif(not BRAVE_DB_URL)` up/down test. `pytestmark = integration`.

**Task 2 — write points (commit 9914b7a)**
- `engine_start` now injects `db: Session = Depends(get_db)` and INSERTs a `RunHistory` row ONLY after the depth 422, source 422, AND `start_run()` success (Pitfall 3 — no phantom rows on rejected starts). Captures `run_id`, persists it to Redis (`brave:engine:run_id`), threads it into `engine_sweep_run.delay(..., run_id=run_id)`. The INSERT is wrapped best-effort so it can never abort a valid start.
- `engine_sweep_run` accepts `run_id`; new `_finalize_run_history()` UPDATEs the row in the `finally` block (ended_at, ufs_dispatched, status `concluido`/`parcial` — parcial detected via the `STOPPING` state read before `mark_idle`). Wrapped in try/except so a finalize write failure never aborts the sweep (T-17.1-02-02). Skipped entirely when `run_id` is None.
- `tests/unit/test_runs_write_path.py` (7 offline tests): no row on 422/422/409, exactly one row + run_id-in-Redis on success, INSERT failure doesn't abort start, finalize swallows a raised write error, finalize skipped when run_id None.

**Task 3 — list + reprocess + register (commit 7edbf8f)**
- `brave/api/routers/runs.py`: typed `RunItem`/`RunsResponse` (extra=forbid). `GET /api/v1/runs` (require_bearer) — source/depth filtered in SQL, uf filtered in Python over the JSON ufs array; `_window_counts()` computes synced (Mar published in window) + failed (Rio dlq/descarte + PoisonQuarantine in window) on-read; total = synced+failed. `PATCH /api/v1/runs/{run_id}/reprocess` (202, require_steward_or_bearer) re-dispatches the run scope via an inline copy of the sweep.py prod-vs-offline broker fallback, audited `run_reprocessed`.
- Registered `runs_router` in `main.py` (additive — dedup router from wave-1 untouched).
- `tests/unit/test_runs_history.py` (8 offline tests): window aggregation, exact envelope shape, uf JSON filtering, pagination, reprocess inline fallback (default + tripadvisor source), 404.
