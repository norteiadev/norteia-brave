# Phase 17.1 Plan 04: Duplicados Frontend Summary

Typed dedup client + mirrored MSW handler + the `PainelDuplicados` view that lists candidate≈Mar pair cards (coincide/diverge chips + labeled similarity) and resolves each via merge/keep/discard against the real backend resolve PATCH.

## What Was Built

**Task 1 — dedup client + MSW handler** (`feat`, commit `c1ff751`)
- `dashboard/lib/dedup-api.ts`: `DedupPairItem` / `DedupPairsResponse` / `DedupDivergedField` / `DedupResolveBody` / `DedupResolveResult` interfaces mirroring the `brave/api/routers/dedup.py` Pydantic models field-for-field (`candidate_id`, `mar_id`, `candidate_rio_id`, `mar_rio_id`, `uf`, `municipio`, `entity_type`, `similarity`, `similarity_source`, `matched_fields`, `diverged_fields`, + `items/total/offset/limit`). `dedupKeys` factory on a shared `['dedup']` prefix. `fetchDedupPairs(uf?)` → `apiFetch("api/v1/dedup/pairs" + qs)`; `resolveDedupPair(candidateRioId, { action, mar_id })` → `apiFetch` PATCH. Relative `api/v1/...` form (the BFF rewrites to `/api/api/v1/...`).
- `dashboard/mocks/handlers/dedup.ts`: handlers registered at the double-prefixed `http://localhost:3000/api/api/v1/dedup/...` path (Pitfall 6). Factories `dedupPairsSuccess()`, `dedupPairsEmpty()`, `dedupPairsError()`, `dedupResolveSuccess()`. Mock payloads typed `satisfies DedupPairsResponse`/`DedupResolveResult`, so the handler IS the field-for-field contract mirror (A5).

**Task 2 — PainelDuplicados view + test** (`feat`, commit `11e0471`)
- `dashboard/components/painel/PainelDuplicados.tsx` (`"use client"`): `useQuery({ queryKey: dedupKeys.pairs(), queryFn: fetchDedupPairs })` loads pairs; one card per item shows candidate (dashed) ≈ Mar (green-accent) panels, a `coincide` chip per `matched_field`, a `diverge` chip per `diverged_field` (candidate/mar values in the chip tooltip), and the labeled similarity (% + `similarity_source` tooltip). Three buttons — Mesclar no existente (merge) / Manter ambos (keep) / Descartar candidato (discard) — wired through `useMutation(resolveDedupPair)` with `toast` + `invalidateQueries(['dedup'])` (PainelTopbar :84-99 scaffold). Yellow validation banner + ✓ empty state. Pure `--painel-*` token styling; green/yellow/red accents use the design's exact oklch literals.
- `__tests__/PainelDuplicados.test.tsx`: renders one card per item with matched/diverged chips + a `95%` similarity; clicking Descartar fires the real resolve PATCH (`/api/api/v1/dedup/pairs/.../resolve`) verified via `server.events.on("request:start", …)`; the empty handler renders the empty state.

## Deviations from Plan

**1. [Rule 3 — Blocking convention] MSW handler not registered in `mocks/handlers/index.ts`**
- **Found during:** Task 1
- **Issue:** The orchestrator note said "register the new handler in `mocks/handlers/index.ts`", but `index.ts` is an intentionally empty barrel (its docblock: "Keeping this default empty means no two slices share a handlers file and the harness boots clean with zero global mocks"). The existing `cost.ts` handler is likewise unregistered; every suite applies its handlers via `server.use(...)`.
- **Fix:** Followed the established codebase pattern — handlers exported from `dedup.ts`, applied per-suite via `server.use(dedupPairsSuccess(), …)`. Registering globally would break the clean-boot offline mandate.
- **Files modified:** none (index.ts left untouched)
- **Commit:** n/a

No other deviations — both tasks executed as written.
