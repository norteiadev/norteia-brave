<objective>
Fix the two field-mask bugs in RealPlacesClient that cause live 400 errors and empty responses, and add addressComponents-to-município extraction so downstream agents receive populated municipio_nome/municipio_ibge fields.

Purpose: RealPlacesClient.text_search currently sends no X-Goog-FieldMask → live 400 INVALID_ARGUMENT. RealPlacesClient.place_details sends metadata with a 'places.' prefix that is wrong for get_place → fields returned empty/default. Both bugs prevent any real Places data from flowing through the pipeline. Fixing them unblocks targeted discovery (Plan 07-03) and the load-test harness (Plan 07-05).

Output: brave/clients/places.py with two field-mask constants, corrected metadata calls, module-level address-component extractor and IBGE lookup builder, and municipio fields populated in text_search results. tests/unit/clients/test_real_places_client.py with 5 offline tests (T1–T5) covering the guard, mask strings, municipio mapping, and publish_time conversion.
