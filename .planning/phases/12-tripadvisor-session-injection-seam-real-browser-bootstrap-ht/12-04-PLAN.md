<objective>
Wire sweep_tripadvisor for fail-fast behaviour on missing/expired sessions, and add a
minimal session-health indicator to the EngineControl dashboard panel.

Purpose: Without this plan, a missing session causes sweep_tripadvisor to retry 3 times
before quarantining — silently ingesting 0 records each time. The fail-fast + dashboard
visibility converts a silent failure into an explicit operator signal: "session expired,
run ta_bootstrap again."

Output: sweep_tripadvisor catches SessionMissingError and SessionExpiredError and stops
cleanly (no retry-storm); Redis carries a needs_bootstrap marker; EngineControl shows a
session-health pill when TripAdvisor source is selected/active.
</objective>
