## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser/operator → BFF → FastAPI /api/v1/workers | Operator Bearer token validated at BFF; service Bearer checked by require_bearer |
| workers.py → Celery broker | Outbound network I/O to Redis broker; must never hang or 500 when broker absent |
| workers.py → Redis (queue depth) | Read-only LLEN; must gracefully return null on connection failure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-06 | Spoofing | GET /api/v1/workers, GET /api/v1/failures | mitigate | require_bearer on both endpoints — constant-time HMAC, fail-closed |
| T-08-07 | DoS (self-inflicted) | celery_app.control.inspect — blocks 1.0s if broker absent | mitigate | timeout=1.0 + try/except wraps entire inspect block; returns 200 with broker_reachable=false; never 500 |
| T-08-08 | Information Disclosure | PoisonQuarantine.payload in failures response | mitigate | payload column NOT serialized in list response — only task_name + error_message (truncated 500 chars) + quarantined_at |
| T-08-09 | Elevation of Privilege | SSRF via inspect() — could attacker redirect broker URL? | accept | inspect() connects to BRAVE_DB_REDIS_URL from config (pydantic-settings); no user-supplied URL; broker URL is operator-controlled env var |
| T-08-SC | Tampering | npm/pip installs | accept | No new packages in this plan |
</threat_model>
