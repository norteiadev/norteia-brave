<domain>
## Phase Boundary

Continuation of CLOSED Phase 17 (slice 1: light shell + Painel/Kanban shipped & verified).
This slice **completes the Painel Brave CMS**: all six views real, and the **destinos+atrativos
synchronization works through the Kanban structure end-to-end**. Two views require NET-NEW backend
(Duplicados, Varreduras); the rest wire to existing endpoints. Lands on the existing `/painel`
route/shell — does NOT touch the 10 legacy dark routes.

**In scope (slice 2):**
1. **Kanban upgrade** — move from 5 to the design's **6 stage columns**: `nascente`, `rio·validação`,
   `whatsapp·contato`, `mar·publicado`, `dlq·revisão`, `falha`. Keep the Destinos/Atrativos metric
   cards, type-filter (Tudo/Destinos/Atrativos), and UF-scope menu. **Full-pipeline drag-drop**:
   a drop fires the REAL backend stage transition for that (entity, target) pair — including
   transitions not supported today (move-backward, force-stage) which require NEW endpoints — with
   safe optimistic revert + toast when a transition is rejected/unavailable.
2. **Duplicados view (NET-NEW backend)** — list dedup-candidate pairs (incoming candidate vs the
   already-published Mar record it resembles, with pgvector cosine similarity + matched/diverged
   fields) and resolve each (merge-into-existing / keep-both / discard-candidate). New FastAPI
   endpoints + Pact consumer contract + React view per design.
3. **Varreduras view (NET-NEW backend)** — persist engine sweep "runs" to a DB table (today they
   live only in Redis), written as the engine runs; list endpoint returns runs by UF/source/depth
   with total/synced/failed counts + status; "↺ Falhas" reprocesses a run's failures. New endpoints
   + Pact + React table view + 7-day stat cards.
4. **Origem modal** — pick source (mtur / tripadvisor / google_places); for TripAdvisor, paste the
   authenticated cURL to (re)establish the session; topbar session pill + expiry toast driven by the
   real TA session TTL. Wire to existing TA session + engine-source endpoints.
5. **Motor on/off toggle** — wire to existing engine start/stop (start requires a depth).
6. **Cross-wire existing views** — Mapeamento (data-mapper), Conversas (WhatsApp, masked phone),
   Custo (spend by lane/model), and the record-edit Drawer (Dados/Conversa tabs) are already built;
   make them all reachable/consistent inside the finished shell with the design's nav (two groups:
   Processamento = Painel/Duplicados/Mapeamento/Varreduras; Operação = Conversas/Custo).

**Core priority:** the Kanban IS the sync surface — promoting a destino/atrativo to **Mar** through
the board must drive the real publish-to-platform path; nothing in the board may invent an endpoint.

**Out of scope:** replacing the 10 legacy dark routes; dark/light theme toggle; new collection
lanes; changing the §7.6 score engine; real external API calls in tests (offline mandate holds).
</domain>

<decisions>
## Implementation Decisions (LOCKED)

### Scope forks (operator-decided 2026-06-27)
- **Duplicados + Varreduras = FULL backend** (not UI-only / not deferred). Build DB migrations,
  FastAPI endpoints, Celery/engine write paths where needed, and Pact consumer contracts. The
  dashboard consumes them through the existing BFF catch-all proxy (`app/api/[...path]/route.ts`)
  with server-side Bearer injection — no per-route BFF files.
- **Kanban drag = FULL pipeline** (not the safe allow-list only). Drops may target any stage,
  including backward moves and force-stage. Backend transitions that don't exist today get NEW
  endpoints. The existing safety boundary stays: a drop that maps to no legal/known transition
  reverts optimistically and toasts "Ação não disponível"; the client NEVER calls an unmapped path.

### Stage / column model (design `support.js` COLS)
- 6 columns, pt-BR labels: `nascente` (Nascente), `rio` (Rio · validação), `whatsapp`
  (WhatsApp · contato), `mar` (Mar · publicado), `dlq` (DLQ · revisão), `falha` (Falha).
- Mapping to backend reality (confirmed this session):
  - routing values on every record: `in_progress` (→ rio), `mar`, `dlq`, `descarte`.
  - atrativos FSM `sub_state`: `discovered`, `contacts_found`, `signals_gathered`,
    **`aguardando_consulta_whatsapp`** (→ whatsapp column), `whatsapp_in_progress`.
  - `nascente` count comes from engine status (not list endpoints); `falha` = error/quarantine
    records (poison/failed sync), distinct from `descarte` (the discard ACTION, reachable via the
    drawer "Descartar" and the board, not a standing column).
- `mapDrop`/`mapRetry` in `lib/painel-actions.ts` is the security boundary — EXTEND it for the new
  full-pipeline transitions; never bypass it.

### Existing wiring to REUSE (do not reinvent)
- Destinos: `GET /destinos`, `/destinos/{id}`, PATCH `promote|descarte|reprocess|edit`.
- Atrativos: `GET /atrativos`, `/atrativos/{id}`, PATCH `advance|descarte|edit`; mar-ready
  `GET /atrativos/mar-ready`, PATCH `/atrativos/{id}/promote`, `POST /atrativos/promote-batch`;
  gate `GET /atrativos/whatsapp-gate`, PATCH `approve|reject`.
- DLQ: `GET /dlq`, `/dlq/{id}`, PATCH `reprocess|validate|descarte`, `POST /dlq/validate-batch`.
- Engine: `GET /engine/status` (live counts + `atrativos_by_sub_state` + UF progress), `POST
  /engine/start` (depth), `POST /engine/stop`. Sweep: `POST /sweep`.
- TripAdvisor: `POST /tripadvisor/session`, `GET /tripadvisor/session/status`,
  `GET /tripadvisor/sweep/progress`.
- Cost: `GET /cost` (by lane/model + window). Conversations: `GET /conversations`,
  `/conversations/{id}` (phone masked via `mask_phone()`). Audit: `GET /audit`.
- Dashboard clients already exist: `lib/destinos-api`, `lib/atrativos-api`, `lib/mar-ready-api`,
  `lib/engine-api`, `lib/cost-api`, `lib/conversations-api`, TA session keys. TanStack Query.

### NET-NEW backend — Duplicados
- Source of truth: `RioRecord.embedding` Vector(1536) (`brave/core/models.py`) + the two-stage dedup
  in `brave/core/rio/dedup.py` (exact content_hash, then territorial-key-blocked pgvector HNSW
  fuzzy search; cosine similarity computed but NOT currently surfaced).
- Add: a way to LIST pending candidate↔Mar pairs (sim score + matched/diverged field labels) and a
  RESOLVE endpoint taking `merge | keep | discard`. Decide persistence shape during research
  (materialized `dedup_pairs` rows vs. computed-on-read from existing embeddings) — research output
  picks the lower-risk option that fits the 100%-offline suite.
- Auth: Bearer/steward-guarded like the rest. Resolution actions audited (reuse audit log).

### NET-NEW backend — Varreduras (runs history)
- Today engine runs live only in Redis (no DB audit trail). Add a persisted `runs_history` table +
  write path (engine/sweep writes a row on start and updates totals/synced/failed/status on
  completion). List endpoint: filter by UF/source/depth; return total/synced/failed + status
  (concluido/parcial/falha). "↺ Falhas" → reprocess that run's failed records (reuse reprocess).

### Frontend / design contract
- Reimplement design markup in React/Next (the `.dc.html` `{{ }}`/`<sc-*>` canvas is reference only).
- Light tokens already scoped at `.painel-light` (`app/globals.css`) — reuse; do NOT flip global dark.
- Geist + Geist Mono already loaded. Exact palette: cream bg `oklch(0.98 0.01 90)`, white `#fff`,
  text `#18181b`, navy `#15315e`, muted `#6b7280`/`#9ca3af`, borders `#e6e4e0`/`#f0eee9`, green
  `oklch(0.55 0.15 150)`, red `oklch(0.55 0.20 27)`, yellow `oklch(0.72 0.15 75)`.

### Research resolutions (2026-06-27, post-RESEARCH.md)
- **A1 — Duplicados similarity:** embeddings are a ZERO-STUB (`compute_embedding` → `[0.0]*1536`,
  `brave/core/rio/dedup.py`), so pgvector cosine is degenerate. Build Duplicados with
  **compute-on-read field/token similarity in Python** (name/município/UF overlap + content_hash),
  matching the design's coincide/diverge chips + a numeric sim. Do NOT materialize a `dedup_pairs`
  table. Implementing real embeddings (cosine) is DEFERRED to a later slice.
- **A2 — merge semantics:** "Mesclar no existente" **unions the candidate's `source_refs`/provenance
  into the existing Mar record and discards the candidate Rio row** (no 409 on differing sources);
  audited via the existing audit log. "Manter ambos" clears the pair (no row change). "Descartar
  candidato" routes the candidate to `descarte`, audited.
- **A5 — contract:** **typed Pydantic response model + mirrored MSW handler** (project's real offline
  pattern). NO new pact-js toolchain. Both sides unit-tested offline.
- **A6 — falha column:** source falha cards from the existing **`GET /api/v1/failures`**
  (PoisonQuarantine) so they are real + draggable for reprocess (consistent with full-pipeline drag);
  if a record cannot be sourced as a card it shows as a count only.
- **Stage transitions:** add **ONE generic, audited `transition` endpoint per entity**
  (destino/atrativo) with a SERVER-SIDE edge allow-list mirroring client `mapDrop`; `mar → *`
  stays 409-blocked (no depublish). Client `mapDrop` is EXTENDED, never bypassed.
- **TA session inject:** no inject client exists today (only status) — add `injectTASession` client
  + wire the Origem modal cURL paste to `POST /tripadvisor/session`.

### Testing (project mandate)
- Backend: pytest, respx for HTTP mocks, NO real Places/OTA/Apify/WhatsApp/OpenRouter/Anthropic/
  norteia-api in CI. Dedup/runs endpoints unit-tested offline; Pact verifies the contract.
- Dashboard: Vitest + MSW; Bearer mocked at network layer. New views get MSW handlers + tests.
- New `@pytest.mark.integration` tests must be skipped-safe without `BRAVE_DB_URL`.
</decisions>
