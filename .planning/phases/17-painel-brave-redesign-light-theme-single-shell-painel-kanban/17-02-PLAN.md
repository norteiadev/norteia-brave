<objective>
Build the Painel data layer: a unified `PainelCard` model + pure, unit-testable selectors that turn the existing destinos/atrativos list endpoints into Kanban columns, a `usePainelBoard` hook that fetches both lists, and a `usePainelMetrics` hook that derives TRUTHFUL per-entity metric counts (from the list envelope `total`, the server-side count — not the loaded array length) plus the real Nascente count from `fetchEngineStatus().counts.nascente`. NO new backend endpoints.

Purpose: Isolate all data shaping in one tested module. Metrics use the envelope `total` (server truth) so "total no escopo" / "progresso %" don't silently undercount; the Nascente column gets a real count even though rio-backed lists don't surface nascente-only records.
Output: `dashboard/lib/painel-data.ts` + its test.
</objective>

    - Test cases: mixed sample (2 destinos: 1 mar 1 dlq; 2 atrativos: 1 in_progress 1 descarte) → buildColumns counts per column; computeMetric(2,1,0) = {total:2,mar:1,falha:0,pct:50}; computeMetric(0,0,0).pct = 0; filterCards by type and by ufs; routingToColumn unknown → "nascente"; duplicate maps from validation_pending; assert no card object contains a phone_e164 key.
