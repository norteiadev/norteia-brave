<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| NorteiaApiClient → norteia-api | Service-to-service boundary; Bearer token is the only auth; token in env var, never logged |
| POST /webhook/error-report | Inbound from norteia-api (or attacker); source_ref must be validated against existing Mar before reopening |
| Pact mock server (pact-python) | Test-local mock; not a real server; no network exposure outside localhost during tests |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | BRAVE_NORTEIA_API_SERVICE_TOKEN leaked in logs | mitigate | token passed in Authorization header only; never logged by structlog (httpx default log does not include headers); .env in .gitignore |
| T-03-02 | Tampering | Unauthenticated webhook reopen attack | mitigate | Phase 1: rate-limit per IP (Plan 02, 10/min Redis); Phase 2 enhancement: add HMAC shared-secret validation between norteia-api and collector. Validated source_ref must exist in Mar (404 on miss, no information leak on valid source_refs). |
| T-03-03 | Spoofing | Pact mock server mistaken for real norteia-api in CI | accept | pact-python mock binds to localhost:* only; mock URL is passed explicitly to NorteiaApiClient; no env var confusion possible |
| T-03-04 | Information Disclosure | Full Mar payload (with PII in canonical fields) sent to norteia-api | mitigate | Phase 1 has no PII in Destinos canonical (name, uf, municipio only); Phase 3 atrativo fields are reviewed against LGPD minimization before adding to canonical; Pact contract payload shape reviewed during Plan 03 freeze |
| T-03-05 | Denial of Service | Retry storm on norteia-api 5xx | mitigate | tenacity stop_after_attempt(3) + wait_exponential(min=2, max=10); Celery max_retries=3; cost guard on LLM (not on norteia-api push, which is cheap per call) |
| T-03-SC | Tampering | pip/uv installs | mitigate | No new packages in this plan; all packages already installed via Plan 01 checkpoint |
</threat_model>
