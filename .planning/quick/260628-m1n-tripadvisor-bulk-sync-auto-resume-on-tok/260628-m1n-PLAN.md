<objective>
Add defense-in-depth auto-resume to the TripAdvisor bulk national sweep so it
recovers automatically from session-token expiry without operator intervention.

Purpose: Currently the sweep halts in stopped_needs_bootstrap and stays there
until an operator manually runs scripts/ta_bulk_sweep.py. Two independent
triggers (inject hook + 60s beat) call one idempotent, race-safe helper that
re-dispatches sweep_tripadvisor.delay once a fresh session is present.

Output:
  - sweep_progress.py: RESUMING state, claim_resume (state-check then SETNX atomic), get_resume_params,
    is_paused_needs_bootstrap, start() stores depth/geo_id/target_max_pages
  - resume.py: maybe_resume_bulk_sweep (single idempotent helper; self-heals on dispatch failure)
  - pipeline.py: bulk branch seeds params into start(); new brave.ta_resume_watch task
  - tripadvisor_session.py: inject hook + TASweepProgressResponse "resuming" state
  - beat_schedule.py: ta-resume-watch 60s entry
  - 4 test files (TDD RED first, then GREEN)
</objective>
