    1. SOURCE — TripAdvisor public website. GraphQL persisted-query endpoint: https://www.tripadvisor.com/data/graphql/ids. No API key required; anti-bot session bootstrap via Playwright + DataDome.

    2. LEGAL RISK — TripAdvisor's Terms of Service (Section 5, "Use of Site") explicitly prohibits systematic scraping, automated data collection, and reproduction of content without permission. Using this source violates TA ToS and may result in IP bans, legal threats, or account suspension. This is a documented, accepted risk under the following conditions.

    3. MITIGATIONS — (a) Low request rate: one UF per minute maximum, no continuous crawl, operator-triggered only. (b) Residential proxy seam: config.proxy_url shields the origin IP from datacenter blocking. (c) No PII stored: only aggregate fields (review_count, rating, most_recent_review_at) are persisted — never author name, reviewer ID, or review text. (d) Operator-gated: sweep is NOT on the autonomous Celery beat; requires explicit /engine/start with source="tripadvisor". (e) Human promote-override: no automated Mar push; every TA attraction requires a human steward action to reach Mar.

    4. LGPD BASIS — Legal basis: Legitimate interest (Art. 7º, IX, LGPD) in territorial data quality for the Norteia platform. Data minimization applied: only aggregate review signals (count, rating, recency) are stored; no individual reviewer data. The LGPD basis applies to the aggregate signals, not to any individual's data. TripAdvisorReviewSignals schema enforces extra="forbid" to prevent any future drift toward PII.

    5. OPERATOR GATE — To run a TripAdvisor sweep: set RUN_REAL_EXTERNALS=1 (or run_real_externals=true in config) and install the optional scraper dep group (pip install 'norteia-brave[scraper]' && playwright install chromium). Sweep is started via POST /api/v1/engine/start {"source": "tripadvisor", "depth": "..."}. It does NOT run in CI and is NOT scheduled on the beat.
