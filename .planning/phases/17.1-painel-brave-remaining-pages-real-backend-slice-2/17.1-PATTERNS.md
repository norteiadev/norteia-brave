**Resolve endpoint + audit choke-point** (copy `descarte_destino` posture, `cms.py:335-378`):
```python
class ResolveBody(BaseModel):
    action: Literal["merge", "keep", "discard"]
    mar_id: uuid.UUID

@router.patch("/api/v1/dedup/pairs/{candidate_rio_id}/resolve",
              dependencies=[Depends(require_steward_or_bearer)])
def resolve_pair(candidate_rio_id: uuid.UUID, body: ResolveBody, db: Session = Depends(get_db)):
    rio = db.get(RioRecord, candidate_rio_id)
    if rio is None:
        raise HTTPException(404, "RioRecord not found")
    before = {"routing": rio.routing}
    if body.action == "discard":
        rio.routing = "descarte"; rio.dlq_reason = "dedup_discarded"; action = "dedup_discarded"
    elif body.action == "merge":
        from brave.core.mar.service import promote_to_mar      # lazy import (cms.py:276 pattern)
        rio.routing = "mar"; promote_to_mar(db, rio); action = "dedup_merged"
    else:  # keep ŌĆö the audit row IS the suppression marker (A3, no schema change)
        action = "dedup_kept"
    write_audit(session=db, action=action, entity_type=rio.entity_type,
                record_id=rio.id, before_state=before,
                after_state={"routing": rio.routing}, actor="steward")
    db.commit()
    return {"status": "ok", "action": body.action}
```

### Audit trail
**Source:** `brave/observability/audit.py:write_audit` (`:22-74`).
**Apply to:** every dedup resolve, every transition, the run finalize/reprocess.
```python
write_audit(session=db, action="<verb>", entity_type=rio.entity_type,
            record_id=rio.id, before_state={...}, after_state={...}, actor="steward")
db.commit()
```
Identical posture to `cms.py:281-296` and `:368-377`. `write_audit` flushes + emits a PII-free structlog line; commit happens in the endpoint.

### Broker-down dispatch fallback (prod-vs-offline)
**Source:** `sweep.py:_dispatch` (`:31-54`) and `engine.py:132-146`.
**Apply to:** runs reprocess, any new task dispatch. Swallow the broker error and run inline ONLY when `AppConfig().run_real_externals is False`; otherwise 503. Keeps the offline suite green without a broker.

### BFF + client + invalidation (frontend)
**Source:** `lib/api-client.ts:apiFetch` (`:55-75`) + `bff()` (`:34-38`); TanStack `useMutation`/`invalidateQueries` (`painel-actions.ts:138-152`, `PainelTopbar.tsx:84-99`).
**Apply to:** all new clients/views. Relative `api/v1/...` path; operator Bearer auto-attached; no new BFF route files (catch-all proxy, CONTEXT-locked). After a mutation, invalidate the resource key (+ `["engine","status"]` for board moves).
