<objective>
Implement the session injection endpoint (POST /api/v1/tripadvisor/session) with its
synchronous canary validation gate, and the session status read endpoint
(GET /api/v1/tripadvisor/session/status).

Purpose: These endpoints are the operator's interface to inject a real browser session
into the worker's Redis slot. The canary gate immediately validates the session
through the production httpx path so an invalid session fails fast at inject time,
not mid-sweep.

Output: brave/api/routers/tripadvisor_session.py with both endpoints; brave/api/main.py
updated to register the router; offline fakeredis test suite covering valid, malformed,
canary-fail, empty-canary-result, and all three GET status reason paths.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| POST /tripadvisor/session body | Operator-injected cookies + queryIds cross the API boundary into Redis |
| Redis BRAVE_TA_SESSION_KEY | Stored session available to any worker that can read this Redis key |
| Redis brave:ta:needs_bootstrap | Read-only flag exposed via GET /session/status; no secrets |
| Canary gate → TripAdvisor | Real httpx call made with operator cookies; response not logged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-02-01 | Information Disclosure | POST body — DataDome cookie values logged | mitigate | Audit log records only cookie_count + query_ids keys; cookie values NEVER appear in structlog calls or error messages; validated by test asserting no "datadome" or cookie value in log output |
| T-12-02-02 | Elevation of Privilege | POST /session — unauthenticated inject | mitigate | require_steward_or_bearer on both endpoints; same constant-time guard as /engine/start; 401 before body is read |
| T-12-02-03 | Denial of Service | Oversized POST body flooding Redis or parsing CPU | mitigate | 64 KB content-length check before Pydantic parse; FastAPI default max body size also applies |
| T-12-02-04 | Spoofing | Canary result faked by a network attacker | accept | Canary runs on the server's own httpx stack; a MITM would need to intercept the server's outbound connection — out of scope for this threat model; canary covers stale-session detection, not MITM |
| T-12-02-05 | Tampering | Redis BRAVE_TA_SESSION_KEY overwritten by concurrent inject | accept | Last-write-wins; concurrent inject race is not a security issue (both callers are authenticated operators); worst case: a new valid session replaces an old one |
| T-12-02-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan |
</threat_model>
